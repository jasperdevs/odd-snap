# Security Policy

If you think you found a vulnerability in Yoink, do not file a public issue with exploit details.

## How to report

- Use GitHub's private vulnerability reporting flow if it is available for this repository
- If that is not available, open a brief issue that says it is security-related and keep the details minimal

## What to include

- A short description of the issue
- The affected version or commit
- Clear reproduction steps if you can provide them safely
- Any logs or screenshots that do not expose private data

## Response expectations

Security reports should be triaged before they are discussed publicly.

## Supported versions

Only the current main branch and the latest release are considered in scope.
